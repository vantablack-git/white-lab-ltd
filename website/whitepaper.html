WhiteLab Protocol ($WLAB)

Abstract. WhiteLab is a compliance-oriented token launch operating system on Base. It unifies gated IDO sales, vesting, staking, a weighted governance lock vault, and timelock governance. Max supply is fixed at 1 billion WLAB with documented emissions, fee burns, and transparent on-chain operations. Code is unaudited; mainnet deployment is gated on an external security review.

Problem & Opportunity

Token launches remain fragmented and trust-deficient. WhiteLab standardizes the launch stack: OpenZeppelin Governor, Merkle-gated IDO, vesting schedules, and protocol consoles for verifiable operations.

Architecture

  • WLABToken — ERC-20 + Votes + Permit + compliance controls
  • WLABTokenSale — Seed / Private / Public phases, refunds, claims
  • WLABVesting — Cliff + linear release, revocable schedules
  • WLABStaking — Tiered locks (30–365d), weighted rewards
  • WLABGovernor + Timelock — 48h+ execution delay, 4% quorum
  • WLABLockVault — Weighted governance lock vault (fixed power, no decay) with gauge voting parallel to token votes
  • WLABTreasuryUUPS — Upgradeable treasury behind proxy

Token Utility

Launch fees, staking rewards, governance, liquidity incentives, bridge fees (Phase 2), and programmatic buyback-and-burn from protocol revenue.

Tokenomics

Max supply: 1,000,000,000 WLAB (fixed cap).

TGE circulating: ~88.5M WLAB (~8.85%) from documented unlock buckets.

Staking emissions: 180M WLAB over 60 months (3M/month from pre-allocated pool).

Governance

Proposal threshold 100,000 WLAB delegated. Voting period ~7 days on Base. Critical operations route through Timelock. Admin roles must migrate to Gnosis Safe before mainnet.

Security

Reentrancy guards, SafeERC20 on value transfers, disabled OFT stub by default, invariant tests on sale obligations and staking weights. External audit required before mainnet capital deployment.

Risks

  • Smart contract vulnerabilities (mitigated via audit + bug bounty)
  • Admin key compromise (mitigated via multisig + timelock)
  • Vesting cliff sell pressure (documented in tokenomics)
  • Regulatory uncertainty for token sales in your jurisdiction
